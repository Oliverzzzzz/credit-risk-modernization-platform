# Dashboard Design

The Streamlit dashboard is designed as a portfolio risk review surface for the credit risk modernization platform.

## Dashboard Goals

The dashboard should help stakeholders inspect:

- prediction volume and risk tier distribution
- model evaluation metrics
- global model drivers
- data quality status
- model governance status
- recent prediction activity

## Target Users

- risk analytics stakeholders
- ML engineering reviewers
- technology consulting interviewers
- model governance reviewers

## Views

The dashboard is organized into:

1. Portfolio overview
2. Model performance
3. Explainability
4. Monitoring and data quality
5. Governance

## Design Principle

The dashboard should feel like an internal enterprise analytics tool, not a marketing landing page. It should prioritize dense, readable information, operational metrics, and review workflows.
